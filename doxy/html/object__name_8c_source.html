4  *
5  * As a special case, if \p ob is 0, the function returns 0.

7  * This name is the name under which the object is stored in the muds object table. It is initialised at the creation of the object such that blueprints are named after the file they are compiled from (without the trailing '.c'), and clones receive the name of their blueprint, extended by "#" followed by a unique non-negative number. These rules also apply to virtual objects - the real name/type of virtual objects is ignored.
8  *
9  * The name of an object can be changed with efun::rename_object(), and efun::object_name() will reflect any of these changes.

12  *

16  * }
17  *
18  * @usage{

24  * ~~~
25  *
26  * }
