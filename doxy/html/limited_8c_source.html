15 DESCRIPTION
16 Call the function <fun> with any given <args> as parameters,
17 and execute it with the given runtime limits.
18 
19 After the function exits, the currently active limits are restored.
20 Result of the efun is the result of the closure call.
21 
22 The runtime limits can be given in two ways: as an array (like the
23 one returned from query_limits(), or as a list of tagged values. If
24 the efun is used without any limit specification, all limits are
25 supposed to be 'unlimited'.
26 
27 The limit settings recognize three special values:
28  LIMIT_UNLIMITED: the limit is deactivated
29  LIMIT_KEEP: the former setting is kept
30  LIMIT_DEFAULT: the 'global' default setting is used.
31 
32 For LIMIT_COST, the special values have these meaning:
33  LIMIT_UNLIMITED: at maximum 1 tick is accounted
34  LIMIT_KEEP: LIMIT_COST is set to 0
35  LIMIT_DEFAULT: LIMIT_COST is set to -100
36 
37 The efun causes a privilege violation ("limited", current_object,
38 fun, limits-array).
39 
40 EXAMPLES
41 limited(#'function)
42 
43  --> executes function with no limits at all
44 
45 limited(#'function, ({ 200000 }), "foo")
46  --> executes function with an eval_cost limit of 200000, and
47  calls function as <function>("foo").
48 
49 limited(lambda(0, ({#'function, "foo"})), LIMIT_EVAL, 200000)
50  --> executes function with an eval_cost limit of 200000, and
51  calls function as <function>("foo").
52
