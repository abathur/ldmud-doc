1 initialisation {#driver_concepts_initialisation}
2 ================================================
3 Global variables, like their local counterparts, can be defined
4 with an initial value:
5 
6 ~~~{.c}
7 int * a = ({ 3, 4 });
8 
9 ~~~
10 The initialization value can be any legal LPC expression,
11 including function calls. The code for the initializations is
12 collected in a compiler-generated function `__INIT()` which is
13 called even before the create-hook is applied on the object.
14 
15 During initialization, blueprints and clones are treated
16 slightly differently:
17 
18 Blueprint variables are always all initialized using `__INIT()`.
19 
20 For clones the programmer can select whether the clone's
21 variables should also be initialized with `__INIT()`, or if they
22 should be assigned from the current blueprint values
23 ('shared with the blueprint'). The latter method is useful for
24 example if blueprints and clones shall share arrays or
25 mappings.
26 
27 The selection is performed with the two pragmas
28 'init_variables' and 'share_variables'. The status of this
29 pragma at the point of the first variable definition counts,
30 and is applied to all variables in the program.
31 
32 The clone initialization method is evaluated per-program, i.e.
33 if an inherited program defines 'share_variables' and the child
34 program doesn't, only the inherited variables are initialized
35 from the blueprint, and all others from `__INIT()`.
36 
37 The default setting for the pragma is configured into the
38 driver, but can also be chosen when starting the driver.
39 
40 ## EXAMPLE #
41 
42 For the object:
43 
44 ~~~{.c}
45 inherit "a";
46 int a = 4;
47 int b;
48 
49 ~~~
50 the compiler executes the equivalent of these `__INIT()` functions:
51 
52 ~~~{.c}
53 #pragma share_variables:
54 
55  unknown __INIT()
56  {
57  "a"::__INIT();
58  if (clonep(this_object()))
59  {
60  a = (blueprint of this_object())->a;
61  }
62  else
63  {
64  a = 4;
65  }
66  return 1;
67  }
68 
69 #pragma init_variables:
70 
71  unknown __INIT()
72  {
73  "a"::__INIT();
74  a = 4;
75  return 1;
76  }
77 
78 ~~~
79 In either case the variable 'b' (in fact all variables) are
80 set to '0' as part of the loading/cloning process before the
81 driver performs the specific initialisation.
82 
83 @par warning:Do not call `__INIT()` yourself, overload, or use it directly in
84 any other way. The implementation of the variable
85 initialization may change at any time.
86
