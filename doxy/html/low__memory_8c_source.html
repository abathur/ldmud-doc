2  /**
3  * This efun is called when there is a (potential) low-memory situation.
4  * Two different limits for the amount of allocated memory can be
5  * configured: a soft limit and a hard limit. If the soft limit is
6  * exceeded, the driver calls low_memory() but does nothing else.
7  * If the hard limit is exceeded, the driver calls low_memory() as well,
8  * but initiates a garbage collection directly after the call. The efun
9  * is called as well directly before a user-initiated garbage collection
10  * is started.
11  *
12  * <what> denotes the type of limit which was exceeded:
13  *
14  * - NO_MALLOC_LIMIT_EXCEEDED (0)No limit was exceeded, but a garbage collection was requested by a
15  * call to garbage_collection() and it will begin right afterlow_memory() returns.
16  * - SOFT_MALLOC_LIMIT_EXCEEDED (1)The soft limit was exceeded.
17  * - HARD_MALLOC_LIMIT_EXCEEDED (2)The hard limit was exceeded. A garbage collection will begin right
18  * after low_memory() returns.
19  *
20  * <limit> specifies the numerical value of the limit which was exceeded.
21  * It is 0 in case of NO_MALLOC_LIMIT_EXCEEDED.
22  *
23  * <memory> specifies the amount of allocated memory right now.
24  *
25  * <reservestate> specifies the current state of the memory reserves of
26  * the driver. The states of the 3 different reserves are OR'ed together.
27  *
28  * - USER_RESERVE_AVAILABLE (1)The user reserve is available.
29  * - MASTER_RESERVE_AVAILABLE (2)The master reserve is available.
30  * - SYSTEM_RESERVE_AVAILABLE (4)The system reserve is available.
31  *
32  * This efun might be used to inform users about the garbage collection
33  * and the expected long lag.
34  *
35  * Another possibility is to initiate a garbage collection deliberately
36  * at a suitable time (e.g. during the following night) after the soft
37  * limit was hit.
38  *
39  * @par note:If in a low_memory condition when the memory reserves have been
40  * used already and they could not be re-allocated after a garbage collection
41  * slow_shut_down() is called instead of calling this function again.
42  *
43  * @synopsis{
44  * void low_memory(int what, int limitvalue, int memory, int reservestate)
45  * }
46  *
47  * @history{
48  * introduced (3.3.719)
49  * }
50  *
51  * @see efun::debug_info(), efun::garbage_collection(), master::slow_shut_down(), @ref driver_concepts_memory "memory"
52  **/
53  void low_memory(int what, int limitvalue, int memory, int reservestate);
54 }
