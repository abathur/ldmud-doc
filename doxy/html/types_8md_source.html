LPC/types.md
1 types {#driver_LPC_types}
2 =========================
3 Variables can have the following types:
4 
5 @ref driver_LPC_int "int"
6 An integer.
7 
8 @ref driver_LPC_string "string"
9 A text string.
10 
11 @ref driver_LPC_status "status"
12 @par obsolete:status was planned to be an optimized boolean format, but this was never actually implemented. status does work; however, since it is only an alias for type 'int', just use int.
13 
14 @ref driver_LPC_object "object"
15 Pointer to an object. Objects are always passed by reference.
16 
17 @ref driver_LPC_arrays "arrays"
18 Pointer to a vector of values, which could also be an alist. Arrays take the form ({ n1, n2, n3 }) and may contain any type or a mix of types. Arrays are always passed by reference. Note that the size of arrays in LPC, unlike most programming languages, CAN be changed at run-time.
19 
20 @ref driver_LPC_mappings "mappings"
21 An 'associative array' consisting of values indexed by keys. The indices can be any kind of datatype. Mappings take the form ([ key1: value1, key2: value2 ]). By default, mappings are passed by reference.
22 
23 @ref driver_LPC_closures "closures"
24 References to executable code, both to local functions, efuns and to functions compiled at run-time ("lambda closures").
25 
26 @ref driver_LPC_symbol "symbol"
27 Identifier names, which in essence are quoted strings. They are used to compute lambda closures, e.g. instead of ({..., 'ident, ... }) you can write declare a 'symbol' variable foo, compute a value for it, and then create the closure as ({ ..., foo, ... })
28 
29 @ref driver_LPC_float "float"
30 A floating-point number.
31 
32 @ref driver_LPC_mixed "mixed"
33 A variable allowed to take a value of any type (int, string, object, array, mapping, float or closure).
34 
35 @ref driver_LPC_structs "struct"
36 A collection of values. See @ref driver_LPC_structs "struct".
37 
38 @ref driver_LPC_unions "unions"
39 A range of types, either of which the variable can contain at runtime. See `unions`.
40 
41 All uninitialized variables have the value 0.
42 
43 The type of a variable is really only for documentation. Unless you define #pragma strong_types or rtt_checks, variables can actually be of any type and has no effect at all on the program. However, it's extremely bad style to declare one type but use another, so please try to avoid this.
44
