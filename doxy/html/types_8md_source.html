1 types {#driver_syntax_types}
2 ============================
3 Variables can have the following types:
4 
5 @ref driver_syntax_int "int"
6 An integer.
7 
8 @ref driver_syntax_string "string"
9 A text string.
10 
11 @ref driver_syntax_status "status"
12 @par obsolete: status was planned to be an optimized boolean format, but this was never actually implemented. status does work; however, since it is only an alias for type 'int', just use int.
13 
14 @ref driver_syntax_object "object"
15 Pointer to an object. Objects are always passed by reference.
16 
17 @ref driver_syntax_arrays "array"
18 Pointer to a vector of values, which could also be an alist. Arrays take the form `({ n1, n2, n3 })` and may contain any type or a mix of types. Arrays are always passed by reference.
19 
20 @par note: the size of arrays in LPC <em>can</em> be changed at run-time.
21 
22 @ref driver_syntax_mappings "mapping"
23 An 'associative array' consisting of values indexed by keys. The indices can be any kind of datatype. Mappings take the form `([ key1: value1, key2: value2 ])`. By default, mappings are passed by reference.
24 
25 @ref driver_syntax_closures "closure"
26 References to executable code, both to local functions, efuns and to functions compiled at run-time ("lambda closures").
27 
28 @ref driver_syntax_symbol "symbol"
29 Identifier names, which in essence are quoted strings. They are used to compute lambda closures, e.g. instead of `({..., 'ident, ... })` you can write declare a 'symbol' variable foo, compute a value for it, and then create the closure as `({ ..., foo, ... })`
30 
31 @ref driver_syntax_float "float"
32 A floating-point number.
33 
34 @ref driver_syntax_mixed "mixed"
35 A variable allowed to take a value of any type (int, string, object, array, mapping, float or closure).
36 
37 @ref driver_syntax_structs "struct"
38 A collection of values. See @ref driver_syntax_structs "struct".
39 
40 @ref driver_syntax_unions "union"
41 A range of types, either of which the variable can contain at runtime. See @ref driver_syntax_unions "union".
42 
43 Unknown: `unknown`
44 An unusable (documentation-only) meta-type indicating that a function's return type is undefined.
45 
46 Null: `null`
47 Indicates a function that returns no value.
48 
49 All uninitialized variables have the value 0.
50 
51 The type of a variable is really only for documentation. Unless you define @ref driver_syntax_pragma "strong_types" or @ref driver_syntax_pragma "rtt_checks", variables can actually be of any type and has no effect at all on the program. However, it's extremely bad style to declare one type but use another, so please try to avoid this.
