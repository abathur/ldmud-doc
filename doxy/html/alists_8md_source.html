1 alists {#driver_obsolete_alists}
2 ================================
3 Alists provide a fast and convenient way to access data
4 associatively.
5 
6 Alists are implemented as arrays of arrays, the first being
7 the array holding the keys, the others arrays holding
8 associated data. An empty alist is an array of empty arrays.
9 
10 Note that the the dimensions of the arrays are used the other
11 way than in lisp to allow for faster searching.
12 
13 Keys have to be of type integer, string or object. Types can
14 be mixed.
15 
16 The search functions return an undefined value when another
17 list is given in place of a presorted key list.
18 
19 A list with non-numeric keys retrieved by restore_object() has
20 to be readjusted by using order_alist(), especially after
21 reboot.
22 
23 Deleting an entry can safely be done with exclude_array as
24 long as all associated data lists are treated like the key
25 array; index finding for such purposes can be done with assoc.
26 
27 Typical applications: holding administrary information about
28 wizards, list of visitors in a pub, list of customers having
29 some sort of credit, information remembered about items etc.
30 
31 @par note:The main use of alists, storing data associatively, is now
32 better performed by mappings. Alists are needed for more
33 extreme situations only.
34 
35 Alists are available only if the driver is compiled with
36 alist support. In that case, __ALISTS__ is defined.
