11 DESCRIPTION
12 Set the max number of commands the interactive <obj> is
13 allowed to execute per second to <num>. A negative value means
14 'unlimited' and is the setting for newly created connections.
15 
16 If <obj> is not given, the current interactive is queried.
17 
18 The function raises a privilege_violation ("set_max_commands",
19 obj, num). If the privilege is denied, the call is ignored.
20 
21 A 'command' in this context means every received data packet
22 which causes a LPC call - actions and calls to input_to()
23 alike.
24 
25 EXAMPLES
26 To establish a mud-wide default for the maximum command rate,
27 write master::connect() like this:
28 
29 object connect() {
30  object obj;
31 
32  ...
33  set_max_commands(100);
34  return obj;
35 
36 }
37 
38 Upon return from the the function, the connection with its
39 modified max_commands will be rebound from the master object
40 to the returned object.
41 
42 Introduced in LDMud 3.2.10.
