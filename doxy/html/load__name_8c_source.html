12 Return the load name for the object <obj> which may be given
13 directly or by its name.
14 
15 If <obj> is a clone, return the load_name() of <obj>'s blueprint.
16 If <obj> is a blueprint, return the filename from which the
17 blueprint was compiled.
18 
19 If <obj> is given by name but not/no longer existing, the
20 function synthesizes the load name as it should be and returns
21 that. If the given name is illegal, the function returns 0.
22 
23 As a special case, if <ob> is 0, the function returns 0.
24 
25 For virtual objects this efun returns the original load_name of the
26 object created by the virtual compiler.
27 
28 If <obj> is omitted, the name for the current object is returned.
29 
30 In contrast to the object_name(), the load name can not be changed
31 by with rename_object() or a VC. However, if an object uses
32 replace_program() the load name no longer reflects the actual
33 behaviour of an object.
34 
35 The returned name starts with a '/', unless the driver is running
36 in COMPAT mode.
37 
38 EXAMPLES
39 object o;
40 o = clone_object("/std/thing");
41 write(load_name(o)); --> writes "/std/thing" in !compat mode
42 
43  and "std/thing" in compat mode
44 
45 write(load_name("/std/thing")); --> same as above
46 write(load_name("/std/thing#4n5")); --> writes 0
47
