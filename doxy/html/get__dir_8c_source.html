12 This function takes a path as argument and returns an array of file
13 names and attributes in that directory.
14 
15 Returns 0 if the directory to search in does not exist.
16 
17 The filename part of the path may contain '*' or '?' as wildcards:
18 every '*' matches an arbitrary amount of characters (or just itself),
19 a '?' matches any character. Thus get_dir("/path/<em>") would return an
20 alphabetically sorted array of all files in directory "/path", or
21 just ({ "/path/</em>" }) if this file happens to exist.
22 
23 To query the content of a directory, use the directory name with a
24 trailing '/' or '/.', for example get_dir("/path/."). Use the
25 directory name as it is to get information about the directory itself.
26 
27 The optional second argument mask can be used to get
28 information about the specified files.
29 
30 GETDIR_EMPTY (0x00) get_dir returns an empty array (not very
31  useful).
32 
33 GETDIR_NAMES (0x01) put the alphabetically sorted file names into
34  the returned array.
35 
36 GETDIR_SIZES (0x02) put the file sizes unsorted into the returned
37  array. directories have size FSIZE_DIR (-2).
38 
39 GETDIR_DATES (0x04) put the file modification dates (in seconds
40  since 01/01/1970) unsorted into the
41  returned array.
42 
43 GETDIR_ACCESS (0x40) put the file access dates unsorted into
44  the returned array.
45 
46 GETDIR_MODES (0x80) put the unix file modes unsorted into
47  the returned array.
48 
49 GETDIR_ALL (0xDF) Return all.
50 
51 GETDIR_PATH (0x10) if this mask bit is set, the filenames with
52  the full path will be returned
53  (GETDIR_NAMES is implied).
54 
55 GETDIR_UNSORTED (0x20) if this mask bit is set, the result of will
56  _not_ be sorted.
57 
58 Note: You should use GETDIR_NAMES|GETDIR_UNSORTED to get the entries
59 in the same order as with GETDIR_SIZES and GETDIR_DATES.
60 
61 The values of mask can be added together.
62 
63 NOTES
64 The path argument to this efun is processed by valid_read() in the
65 mudlib master before being used. The mudlib may normalize this path
66 (e.g. strip leading or trailing "/") and this may lead to expected
67 results (e.g. get_dir("/dir/", ...) not returning the contents
68 of /dir/).
69 COMPAT mode: GETDIR_PATH will return the paths without leading /.
