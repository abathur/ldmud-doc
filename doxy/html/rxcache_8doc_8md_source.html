1 rxcache.doc {#driver_obsolete_rxcache.doc}
2 ==========================================
3 The rxcache acts as a wrapper around the regcomp() function
4 in the regexp module. It keeps track of the recently compiled
5 regular expression and stores both the expression string and
6 regexp's internal representation. If an expression is found
7 among the stored ones, the rxcache returns the stored internal
8 representation compiled earlier.
9 
10 The interface is through the function regcomp_cache() which
11 takes the same parameters as regcomp(). However, since the
12 compiled expression is stored and managed by rxcache, the
13 caller of regcomp_cache() must not deallocate the compiled
14 expression after use.
15 
16 The expressions are stored in a fixed size table, with entries
17 indexed by a hash over the pattern string. The table holds
18 RXCACHE_TABLE entries, and every entry can hold RXCACHE_CHAIN
19 expressions. Upon a hash collision, one of the stored expressions
20 is removed (and deallocated) using a round-robin selection.
21 This behaviour may not be optimal, but on the other hand
22 eventually deallocates expressions which are no longer used.
23 The two >>RXCACHE_<< values are defined in config.h.
24 
25 Initial tests showed an acceleration of the regexp _setup_
26 phase by factor 4. However, since the actual matching is as
27 fast as before, the net savings will be lower.
28 
29 To install the rxcache, apply the diff file (use parameter
30 '-p1' with patch) and reconfigure and recompile the driver.
