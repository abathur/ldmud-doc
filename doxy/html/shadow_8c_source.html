8  object shadow(object ob, int flag);
9 }
10 If <flag> is non-zero the current object will shadow <ob>. If
11 <flag> is 0, either 0 or the object that is shadowing <ob>
12 will be returned.
13 
14 The calling object must be permitted by the master object to
15 do the shadowing. In most installations, an object that
16 defines the function query_prevent_shadow() to return 1
17 can't be shadowed, and the shadow() function will return 0
18 instead of ob.
19 
20 shadow() also fails if: the calling object tries to shadow
21 a function that was defined as "nomask"; the program was
22 compiled with the #pragma no_shadow; or the calling
23 object is already shadowing, being shadowed, or has an
24 environment. Also, the target <ob> must not be shadowing
25 another object.
26 
27 If an object A shadows an object B, every call_other() to B
28 will be redirected to A. If object A has not defined the
29 function, the call will be passed to B. There is only one
30 object that can directly call functions in B with call_other(),
31 and that is A. Not even object B can call_other() itself; the
32 calls will still be redirected to object A. All normal (internal)
33 function calls inside B will, however, remain internal to B.
