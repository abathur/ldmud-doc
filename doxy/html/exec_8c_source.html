10 exec() switches the connection from the interactive object old
11 to the object new. If the new object is also interactive, it's
12 connection will be transferred to the old object, thus
13 exchaning the two connections between the object. If the new
14 object is not interactive, the old will not be interactive
15 anymore after the exec call succeeded.
16 
17 The result is 1 on success, and 0 on failure.
18 
19 exec() is used to load different "user objects" or to reconnect
20 link dead users.
21 
22 To provide security mechanisms, the interpreter calls
23 master->valid_exec(current_program, new, old), which must
24 return anything other than 0 to allow this exec() invocation.
25 
26 After the exec(), if arg 2 was this_player(), this_player()
27 becomes arg 1, else vice versa. Ditto for this_interactive().
28 
29 Take care when writing a simul-efun around exec(): the
30 'current_program' passed to the valid_exec() function will be
31 that of the simul-efun object. To get around this, use
32 bind_lambda() to bind #'exec to the real object and funcall()
33 the resulting closure.
