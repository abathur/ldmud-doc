17 Call the function <ob>-><func>() resp. the closure <cl> for
18 every element of the array, or mapping <arg>, and return a
19 result made from those elements for which the function call
20 returns TRUE. The <extra> arguments are passed as additional
21 parameters to the function calls and must not be references of
22 array of mapping elements (like &(i[1]) ).
23 
24 If <ob> is omitted, or neither a string nor an object, it
25 defaults to this_object().
26 
27 If <arg> is an array or struct, the function will be called with
28 each of the array/struct values as first parameter, followed by the
29 <extra> arguments. If the result from the function call is true,
30 the array element in question is included into the efun result.
31 
32 If the efun is used with a mapping <map> instead of a function,
33 every array element which is key in <map> is included into the
34 result.
35 
36 If <arg> is a mapping, the function will be called with
37 each of the mapping keys as first, and (if existing) the
38 associated values as second parameter, followed by the <extra>
39 arguments. If the result is true, the mapping element in question
40 is included into the result.
41 
42 Depending on the width of the mapping <arg>, the function
43 call takes one of three forms:
44 
45  widthof(arg) == 0: ob->func(key, 0, extra...)
46  == 1: ob->func(key, arg[key], extra...)
47  > 1: ob->func( key
48  , ({ arg[key,0] ...arg[key,width-1] })
49  , extra...)
50 
51 The advantage of this approach is that the two types of
52 multi-dimensional mappings (mappings with multiple values
53 per key, and mappings of arrays) can be treated in the same way.
54 
55 Historical Note: filter() used with arrays behaves like
56 filter_array(), but used with mappings is a generalisation of
57 filter_indices().
