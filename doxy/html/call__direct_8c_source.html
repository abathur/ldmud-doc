1 public class efun {
2  /**
3  * Call a member function \p fun in another object \p ob with argument(s) \p arg... . Result is the value returned from the called function (or 0 for non-existing or void functions).
4  *
5  * This efun is a twin to efun::call_other(), with the difference being that efun::call_direct() never calls a default method.
6  *
7  * Additionally the efun accepts an array of objects as \p ob: the function is called with the same arguments in all the given objects. The single results are collected in an array and yield the final result. Array elements can be objects or the names of existing objects; destructed objects and 0s will yield a '0' as result, but don't cause an error.
8  *
9  * The object(s) can be given directly or via a string (i.e. its object_name). If it is given by a string and the object does not exist yet, it will be loaded.
10  *
11  * `ob->fun(args)` and `"ob_name"->fun(args)` are equivalent to `call_other(ob, "fun", args)`. Nowadays the `ob_name` string can also be a variable.
12  *
13  * `ob->fun(args)` and `ob->"fun"(args)` are equivalent to `call_other(ob, "fun", args)`. `ob->(fun)(args)` are equivalent to `call_other(ob, fun, args)` where fun is a runtime expression returning the function name.
14  *
15  * If ob::fun does not define a publicly accessible function, the efun::call_other() will return 0, which is indistinguishable from a function returning 0.
16  *
17  * "publicly accessible" means "public" when calling other objects, and "public" or "static" when calling `this_object()`. "private" and "protected" function can never be called with efun::call_other().
18  *
19  * The return type of efun::call_other() is `any` by default. However, if your LPC code uses @ref driver_syntax_pragma "strict_types", the return is @ref driver_syntax_types "unknown", and the result of efun::call_other() must be casted to the appropriate type before you can use it for anything.
20  *
21  * @synopsis{
22  * unknown call_direct(object ob, string fun, mixed arg, ...)
23  * unknown call_direct(object *ob, string fun, mixed arg, ...)
24  * }
25  *
26  * @usage{
27  * All the following statements call the lfun `QueryProp()` in the current player with the argument `P_SHORT`:
28  *
29  * ~~~{.c}
30  * string str, fun;
31  *
32  * str = (string)call_direct(this_player(), "QueryProp", P_SHORT);
33  * fun = "QueryProp";
34  * str = (string)call_direct(this_player(), fun, P_SHORT);
35  *
36  * You have to do explicit type casting because of the unknown
37  * return type, if you have set #pragma strict_types.
38  *
39  * // This statement calls the lfun short() in all interactive users
40  * // and stores the collected results in a variable.
41  * string *s;
42  *
43  * s = (string *)call_direct(users(), "short");
44  *
45  * ~~~
46  *
47  * }
48  *
49  * @history{
50  * introduced (3.3.113) -- with the H_DEFAULT_METHOD.
51  * changed (3.2.10) -- made the call on arrays of objects configurable.
52  * changed (3.5.0) -- the call on arrays of objects non-optional.
53  * }
54  *
55  * @see efun::call_other(), efun::call_direct_resolved(), efun::call_resolved(), applied::create(), @ref driver_syntax_pragma "pragma", efun::extern_call(), efun::function_exists(), @ref driver_syntax_functions "functions", efun::map_objects()
56  **/
57  unknown call_direct(object *ob, string fun, mixed arg, ...);
58 }
