11 Call a member function <fun> in another object <ob> with
12 argument(s) <arg...>. Result is the value returned from
13 the called function (or 0 for non-existing or void functions).
14 
15 This efun is a twin to call_other(), with the difference
16 being that call_direct() never calls a default method.
17 
18 Additionally the efun accepts an array of objects as <ob>: the
19 function is called with the same arguments in all the given objects.
20 The single results are collected in an array and yield the final
21 result. Array elements can be objects or the names of existing
22 objects; destructed objects and 0s will yield a '0' as result, but
23 don't cause an error.
24 
25 The object(s) can be given directly or via a string (i.e. its
26 object_name). If it is given by a string and the object does not
27 exist yet, it will be loaded.
28 
29 ob->fun(args) and "ob_name"->fun(args) is equivalent to
30 call_other(ob, "fun", args). Nowadays the ob_name string can
31 also be a variable.
32 
33 ob->fun(args) and ob->"fun"(args) are equivalent to
34 call_other(ob, "fun", args). ob->(fun)(args) are equivalent
35 to call_other(ob, fun, args) where fun is a runtime expression
36 returning the function name.
37 
38 If ob::fun does not define a publicly accessible function, the
39 call_other() will return 0, which is indistinguishable from
40 a function returning 0.
41 
42 "publicly accessible" means "public" when calling other objects,
43 and "public" or "static" when calling this_object(). "private"
44 and "protected" function can never be called with call_other().
45 
46 The return type of call_other() is 'any' be default. However,
47 if your LPC code uses #pragma strict_types, the return type is
48 'unknown', and the result of call_other() must be casted to
49 the appropriate type before you can use it for anything.
50 
51 EXAMPLES
52 // All the following statements call the lfun QueryProp()
53 // in the current player with the argument P_SHORT.
54 string str, fun;
55 
56 str = (string)call_direct(this_player(), "QueryProp", P_SHORT);
57 fun = "QueryProp";
58 str = (string)call_direct(this_player(), fun, P_SHORT);
59 
60 You have to do explicit type casting because of the unknown
61 return type, if you have set #pragma strict_types.
62 
63 // This statement calls the lfun short() in all interactive users
64 // and stores the collected results in a variable.
65 string >>*<<s;
