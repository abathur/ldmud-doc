10 DESCRIPTION
11 This efun is for backward compatibility only. It is only
12 available in compat mode.
13 
14 Move the object "item" to the object "dest". All kinds of
15 tests are done, and a number is returned specifying the
16 result:
17 
18 0: Success.
19 1: Too heavy for destination.
20 2: Can't be dropped.
21 3: Can't take it out of it's container.
22 4: The object can't be inserted into bags etc.
23 5: The destination doesn't allow insertions of objects.
24 6: The object can't be picked up.
25 
26 If an object is transfered to a newly created object, make
27 sure that the new object first is transfered to its
28 destination.
29 
30 The efun calls add_weight(), drop(), get(), prevent_insert(),
31 add_weight(), and can_put_and_get() where needed.
32
