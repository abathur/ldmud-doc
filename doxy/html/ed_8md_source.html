1 ed {#driver_LPC_ed}
2 ===================
3 The @ref driver_LPC_ed "ed" command opens a text editor. Ed has two modes, command mode and insert mode. Command mode uses `:` for a prompt, while insert mode has no prompt. You exit insert mode by typing a single `.` on a line.
4 
5 @par note:These manpages seem a bit antique, though still correct. For a better detailed help, invoke ed and use the `h` command. Also you could look into the man page for ed(1) on your nearest Unix box. And for a bit of fun you can try the good old quiz(6), invoke as `quiz function ed-command`.

7 ## SYNTAX FORMS #
8 
9 All commands entered in either mode use one of the following syntax forms. With a line range:
10 
11 ~~~{.c}
12 <X>,<Y><cmd>
13 
14 ~~~
15 With a single line:
16 
17 ~~~{.c}
18 <X><cmd>
19 
20 ~~~
21 Or with no line specified:
22 
23 ~~~{.c}
24 <cmd>
25 
26 ~~~
27 To print lines 1 to 10, fill the first form with `1` and `10` and `p`:
28 
29 ~~~{.c}
30 1,10p
31 
32 ~~~
33 To delete lines 1 to 5, fill the first form with `1`, `5` and `d`:
34 
35 ~~~{.c}
36 1,5d
37 
38 ~~~
39 To print a single line, fill the second form with `8` and `p`:
40 
41 ~~~{.c}
42 8p
43 
44 ~~~
45 A `.` is the "current line". The current line is the last line referenced. If you want to print last line + 10 more:
46 
47 ~~~{.c}
48 .,.+10p
49 
50 ~~~

81

84

87

90

102 ## SUBSTITUTIONS #
103 
104 Substitutions are very advanced.
105 
106 First a simple example:
107 
108 `s/apa/bepa/`
109 This will substitue the `apa` in current line with `bepa`. If `p` is appended, you will also immediately see the result.
110 
111 `1,$s/apa/bepa/`
112 Same, but all lines in file. Only first occurence on every line.
113 
114 Any character can used instead of `/`:
115 
116 ~~~{.c}
117 s!apa!bepa!g
118 
119 ~~~
120 The `g` specifies that all occurences of apa on this line are changed to bepa, instead of just the first.
121 
122 The replacement pattern can be a regular expression, as covered in the next section.

124 ## SEARCHING #
125 
126 Searching is done with:
127 
128 ~~~{.c}
129 /hello/
130 
131 ~~~
132 This will find first match in or after current line. To repeat the search, use:
133 
134 ~~~{.c}
135 //
136 
137 ~~~
138 The patterns are regular expressions, where some characters have a special meaning:
139 
140 `.`
141 Match any character.
142 
143 `x*`
144 Match any numbers of x (0 or more).
145 
146 `[abc]`
147 Match `a`, `b` or `c`.
148 
149 `[0-9]`
150 Match any digit 0 - 9.
151 
152 `[a-z]`
153 Match any lowercase letter.
154 
155 `\x`
156 Match `x` where `x` can be any character except `(` and `)`.
157 
158 @par note:This only half the truth, there is lots more magic in the regexps.

160 ## COPYING #
161 
162 How to copy from a standard file.
163 
164 Enter ed. Then do `r /room/vill_green.c`. Now you have something in the buffer. Change it into what you want it to be. Then `w /players/peter/hall.c`. Or `w hall.c`.

257 ## TIPS & TRICKS #
258 
259 1. never use `1,$p` to print out an editfile, because you will be thrown out 'cause of too much text transfereed to you.
260 2. `$`: jump to end of file.
261 3. `?anything?` and `??` : search from bottom to up. (like '/' from beginning to end of file. (also with substitutions, try it out...)
262 4. ( `g/xxx/p` search global xxx and print corresponding lines, `/xxx/s/new/old/p` : search xxx, substitute new to old in this line and print out. (try this concatenations with other commands)
263 5. `1,nmx` ( see ed5 ), but also: `1,ntx` : don't move the lines, but make a copy of them.
264 6. `x,y w name` : save lines `x` to `y` to `file` name (if you don't know the line numbers : `=` current line number)
265 7. `s/$/text/p` : append text to the end of current LINE and print line
266 8. `s/^/text/p` : insert text at beginning of current LINE and print line
