2  /**
3  * Validate the execution of a privileged operation. op denotes the requested operation, who is the object requesting the operation (file_name or object pointer), \p arg and \p arg2 are additional arguments, depending on the operation.
4  *
5  * The function should return >0 to grant the privilege, 0 to indicate that the caller was probably misled and the error might be fixed, and anything else to indicate a real violation that will be handled as run time error.
6  *
7  * The privileged operations are:
8  *
9  * attach_erq_demon
10  * Attach the erq demon to object \p arg with flag \p arg2.
11  *
12  * bind_lambda
13  * Bind a lambda-closure to object \p arg.
14  *
15  * call_out_info
16  * Return an array with all call_out informations.
17  *
18  * configure_interactive
19  * Set option \p arg2 with value \p arg3 as default (`arg==0`) or for object \p arg.
20  *
21  * configure_driver
22  * Set option `arg1` to value(s) \p arg2.
23  *
24  * enable_telnet
25  * Enable/disable telnet (\p arg2) for object \p arg.
26  *
27  * execute_command
28  * Execute command string \p arg2 for the object \p arg.
29  *
30  * erq
31  * At the request \p arg2 is to be sent to the erq-demon by the object \p who.
32  *
33  * garbage_collection
34  * Object \p who calls the efun::garbage_collection() with \p arg as filename and \p arg2 as flag.
35  *
36  * input_to
37  * Object \p who redirects the next input from commandgiver \p arg, using \p arg2 as value for the flags. This is used for flag values including the 'no bang' option.
38  *
39  * limited
40  * Execute \p arg with reduced/changed limits \p arg2 (as return by efun::query_limits()).
41  *
42  * mysql
43  * Object \p who attempted to execute mySQL efun \p arg.
44  *
45  * pgsql
46  * Object \p who attempted to execute Postgres efun \p arg.
47  *
48  * net_connect
49  * Attempt to open a connection to host \p arg, port \p arg2.
50  *
51  * nomask simul_efun
52  * Attempt to get an efun \p arg via efun:: when it is shadowed by a nomask type simul_efun.
53  *
54  * rename_object
55  * The object \p who tries to rename the object \p arg to the name \p arg2.
56  *
57  * send_udp
58  * Send UDP-data to host \p arg.
59  *
60  * get_extra_wizinfo
61  * Get the additional wiz-list info for user \p arg.
62  *
63  * set_extra_wizinfo
64  * Set the additional wiz-list info for user \p arg.
65  *
66  * set_extra_wizinfo_size
67  * Set the size of the additional user info in the wiz-list to \p arg.
68  *
69  * set_driver_hook
70  * Set hook \p arg to \p arg2.
71  *
72  * set_limits
73  * Set limits to \p arg (as returned by efun::query_limits()).
74  *
75  * set_max_commands
76  * Set the max. number of commands interactive object \p arg can issue per second to \p arg2.
77  *
78  * set_this_object
79  * Set `this_object()` to \p arg.
80  *
81  * shadow_add_action
82  * Add an action to function \p arg2 of object \p arg from the shadow \p who which is shadowing \p arg.
83  *
84  * shutdown
85  * Object \p who calls the efun shutdown with \p arg as argument.
86  *
87  * sqlite_pragma
88  * Execute pragma statement in SQLite.
89  *
90  * symbol_variable
91  * Attempt to make a symbol from a hidden inherited variable. \p arg is the object in question, \p arg2 the number of the variable in the variable table.
92  *
93  * variable_list
94  * An attempt to return the variable values of object \p arg is made from a different object \p who.
95  *
96  * wizlist_info
97  * Return an array with all wiz-list information.
98  *
99  * efun::call_out_info() can return the arguments to functions and lambda closures to be called by efun::call_out(); you should consider that read access to closures, mappings and arrays means write access and/or other privileges. efun::wizlist_info() will return an array which holds, among others, the extra wizlist field. While a toplevel array, if found, will be copied, this does not apply to nested arrays or to any mappings. You might also have some sensitive closures there. efun::send_udp() should be watched as it could be abused. The xxx_extra_wizinfo operations are necessary for a proper wizlist and should therefore be restricted to admins. All other operations are potential sources for direct security breaches - any use of them should be scrutinized closely.
100  *

103  * }
104  *
105  * @history{

107  * changed (3.3.563) -- added the passing of the limits to the "limited" and "set_limits".
108  * changed (3.2.11/3.3.640) -- added the "mysql" violation.
109  * changed (3.3.717) -- added the "sqlite_pragma" violation.
110  * }
111  *
