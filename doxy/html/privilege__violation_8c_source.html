2  /**
3  * Validate the execution of a privileged operation.
4  * op denotes the requested operation, who is the object
5  * requesting the operation (file_name or object pointer), <arg>
6  * and <arg2> are additional arguments, depending on the operation.
7  *
8  * The function should return >0 to grant the privilege, 0 to
9  * indicate that the caller was probably misled and the error
10  * might be fixed, and anything else to indicate a real
11  * violation that will be handled as run time error.
12  *
13  * The privileged operations are:
14  *
15  * attach_erq_demon
16  * Attach the erq demon to object <arg> with flag <arg2>.
17  *
18  * bind_lambda
19  * Bind a lambda-closure to object <arg>.
20  *
21  * call_out_info
22  * Return an array with all call_out informations.
23  *
24  * configure_interactive
25  * Set option <arg2> with value <arg3> as default (<arg>==0) or for object
26  * <arg>.
27  *
28  * configure_driver
29  * Set option <arg1> to value(s) <arg2>.
30  *
31  * enable_telnet
32  * Enable/disable telnet (<arg2>) for object <arg>.
33  *
34  * execute_command
35  * Execute command string <arg2> for the object <arg>.
36  *
37  * erq
38  * At the request <arg2> is to be sent to the erq-demon by the object <who>.
39  *
40  * garbage_collection
41  * Object <who> calls the efun garbage_collection() with <arg> as filename
42  * and <arg2> as flag.
43  *
44  * input_to
45  * Object <who> redirects the next input from commandgiver <arg>, using
46  * <arg2> as value for the flags. This is used for flag values including the
47  * 'no bang' option.
48  *
49  * limited
50  * Execute <arg> with reduced/changed limits 1<arg2> (as return by
51  * query_limits()).
52  *
53  * mysql
54  * Object <who> attempted to execute mySQL efun <arg>.
55  *
56  * pgsql
57  * Object <who> attempted to execute Postgres efun <arg>.
58  *
59  * net_connect
60  * Attempt to open a connection to host <arg>, port <arg2>.
61  *
62  * nomask simul_efun
63  * Attempt to get an efun <arg> via efun:: when it is shadowed by a nomask
64  * type simul_efun.
65  *
66  * rename_object
67  * The object <who> tries to rename the object <arg> to the name <arg2>.
68  *
69  * send_udp
70  * Send UDP-data to host <arg>.
71  *
72  * get_extra_wizinfo
73  * Get the additional wiz-list info for user <arg>.
74  *
75  * set_extra_wizinfo
76  * Set the additional wiz-list info for user <arg>.
77  *
78  * set_extra_wizinfo_size
79  * Set the size of the additional user info in the wiz-list to <arg>.
80  *
81  * set_driver_hook
82  * Set hook <arg> to <arg2>.
83  *
84  * set_limits
85  * Set limits to <arg> (as returned by query_limits()).
86  *
87  * set_max_commands
88  * Set the max. number of commands interactive object <arg> can issue per
89  * second to <arg2>.
90  *
91  * set_this_object
92  * Set this_object() to <arg>.
93  *
94  * shadow_add_action
95  * Add an action to function <arg2> of object <arg> from the shadow <who>
96  * which is shadowing <arg>.
97  *
98  * shutdown
99  * Object <who> calls the efun shutdown with <arg> as argument.
100  *
101  * sqlite_pragma
102  * Execute pragma statement in SQLite.
103  *
104  * symbol_variable
105  * Attempt to make a symbol from a hidden inherited variable. <arg> is the
106  * object in question, <arg2> the number of the variable in the variable table.
107  *
108  * variable_list
109  * An attempt to return the variable values of object <arg> is made from a
110  * different object <who>.
111  *
112  * wizlist_info
113  * Return an array with all wiz-list information.
114  *
115  * call_out_info() can return the arguments to functions and
116  * lambda closures to be called by call_out(); you should
117  * consider that read access to closures, mappings and arrays
118  * means write access and/or other privileges.
119  * wizlist_info() will return an array which holds, among others,
120  * the extra wizlist field. While a toplevel array, if found,
121  * will be copied, this does not apply to nested arrays or to any
122  * mappings. You might also have some sensitive closures there.
123  * send_udp() should be watched as it could be abused.
124  * The xxx_extra_wizinfo operations are necessary for a proper
125  * wizlist and should therefore be restricted to admins.
126  * All other operations are potential sources for direct security
127  * breaches - any use of them should be scrutinized closely.
128  *
129  * @synopsis{
130  * int privilege_violation(string op, mixed who, mixed arg, mixed arg2, mixed arg3)
131  * }
132  *
133  * @history{

135 "set_max_commands" and "variable_list" violations.
136  * changed (3.3.563) -- added the passing of the limits to the "limited"
137 and "set_limits".
138  * changed (3.2.11/3.3.640) -- added the "mysql" violation.
139  * changed (3.3.717) -- added the "sqlite_pragma" violation.
140  * }
141  *
142  * @see efun::net_connect(), efun::send_erq(), efun::set_this_object(), efun::rename_object(), @ref driver_concepts_simul_efun "simul_efun", efun::call_out_info(), efun::shadow(), efun::add_action(), efun::bind_lambda(), efun::send_udp(), efun::input_to(), efun::execute_command(), efun::variable_list(), efun::enable_telnet(), @ref driver_concepts_mysql "mysql"
