4  *
5  * When efun::set_modify_command() was called, the parser won't expand the standard abbreviations (n,e,s,w,nw,sw,ne,se) for that user anymore, nor use any hook set for this.
6  *
7  * - 0 as argument will stop the command modification and reinstall the standard abbreviations.
8  * - -1 as argument will just return the object previously set.
9  *
10  * The return value is the object that was previously set with efun::set_modify_command(), if any.
11  *
12  * This mechanism is intended to expand aliases on quicktypers or the like. The name of the lfun called can be changed from >>:apply:`modify_command`<< to something else using the @ref driver_hook_modify_command_fname "H_MODIFY_COMMAND_FNAME" hook.
13  *
14  * Up to @subpage driver_mailto:3.2.1@108 "3.2.1@108" the lfun called was hardcoded as 'modify_command'. The hook @ref driver_hook_modify_command_fname "H_MODIFY_COMMAND_FNAME" introduced
15  * in @subpage driver_mailto:3.2.1@109 "3.2.1@109" allows to change the name.
16  *
17  * @synopsis{
18  * object set_modify_command(object)
19  * object set_modify_command(string)
20  * object set_modify_command(int)
21  * }
22  *
23  * @see efun::command(), @ref driver_hook_overview "hook", applied::modify_command()
24  **/
25  object set_modify_command(int);
