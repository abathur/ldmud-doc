12 Tell the parser to initiate a garbage collection after the
13 current execution ended. Depending on the type of memory
14 allocator used, this GC is more less thorough.
15 
16 If the 'smalloc' memory allocator is used, the GC produces
17 output to a log file. The default name of the log file is
18 specified at program start, but can be modified at runtime:
19 
20  With the <filename> argument a log file for the GC output
21  different from the default log file can be specified.
22 
23  If <flag> is not given or 0, the output from the next
24  and only the next GC will go into the log file. If the file
25  already exists, the new output will be appended.
26 
27  If <flag> is 1, the <filename> will be used as the new
28  default log file for all following GCs. If the file already
29  exists, it will be overwritten from the start.
30 
31 If a different memory allocator is used, the GC does not produce
32 output and the <filename> and <flag> arguments are ignored.
33 
34 Calling this efun causes a privilege_violation.
