2  /**
3  * @par note:In LPC syntax `->` is an alias for this efun. Because the syntax form is easier to read, efun::call_other() is rarely used directly in new code. See the syntax form's documentation for usage notes.
4  *
5  * Call a member function <fun> in another object <ob> with an the argument(s) <arg...>. Result is the value returned from the called function (or 0 for non-existing or void functions).
6  *
7  * Additionally the efun accepts an array of objects as <ob>: the function is called with the same arguments in all the given objects. The single results are collected in an array and yield the final result. Array elements can be objects or the names of existing objects; destructed objects and 0s will yield a '0' as result, but don't cause an error.
8  *
9  * The object(s) can be given directly or via a string (i.e. its object_name). If it is given by a string and the object does not exist yet, it will be loaded.
10  *
11  * If ob::fun does not define a publicly accessible function, the efun will call the H_DEFAULT_METHOD hook if set. If the hook is not set or can't resolve the call either, call_other() will return 0, which is indistinguishable from a function returning 0.
12  *
13  * Calls to the master object never use the H_DEFAULT_METHOD hook. To force non-default calls, the efun call_direct() can be used.
14  *
15  * "publicly accessible" means "public" when calling other objects, and "public" or "static" when calling this_object(). "private" and "protected" function can never be called with call_other().
16  *
17  * The return type of call_other() is 'any' be default. However, if your LPC code uses #pragma strict_types, the return type is 'unknown', and the result of call_other() must be casted to the appropriate type before you can use it for anything.
18  *
19  * @par note:You may find calls to unlikely functions, especially in older code. For example:~~~{.c}
20  * !Compat: call_other("/users/luser/thing", "???", 0);
21  * Compat: call_other("users/luser/thing", "???", 0);
22  *
23  * ~~~This looks a bit weird but it was used very often to just load the object by calling a non-existing function like "???". Fortunately nowadays there is an efun load_object() for this purpose.
24  *
25  * @synopsis{
26  * unknown call_other(object ob, string fun, mixed arg, ...)
27  * unknown call_other(object *ob, string fun, mixed arg, ...)
28  * }
29  *
30  * @usage{
31  * First let's do some setup:
32  * string str, fun;
33  * fun = "QueryProp";
34  * Now we'll use these variables with the basic call_other(). Both of the following statements call the lfun QueryProp() in the current player object with a single argument, P_SHORT:
35  * str = (string)call_other(this_player(), "QueryProp", P_SHORT);
36  * str = (string)call_other(this_player(), fun, P_SHORT);
37  * If you have set #pragma strict_types, you have to do explicit type casting because of the unknown return type.
38  * You can also pass call_other() an array of objects, causing the function to be called for each object. The following statement calls the lfun short() in all interactive users and stores the result in an array of strings:
39  * string *s;
40  * s = (string *)call_other(users(), "short");
41  * }
