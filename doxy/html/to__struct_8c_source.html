13 DESCRIPTION
14 The given array, mapping or struct <data> is returned as a struct.
15 If a <template> struct is given, the returned struct is of the same
16 type. Without a template, an anonymous struct is returned in case of
17 arrays and mappings and in case of structs <data> is returned
18 unchanged.
19 
20 If <data> is an array, its elements are assigned in order to the
21 resulting struct. For an anonymous struct, all elements of <data>
22 are assigned, for a templated struct only as many as fit into
23 the struct.
24 
25 If <data> is a mapping and no template is given, the resulting
26 anonymous struct contains all elements from <data> with a string
27 key; the key name is used as struct member name.
28 
29 If <data> is a mapping and a template is given, the struct
30 member names are used as keys for lookups in <data>; the found
31 data is assigned to the struct members.
32 
33 If <data> is a struct and a template is given, a struct of the type
34 of template is created and all members from <data> are copied to the
35 new struct, which exist in both structs. This conversion is only
36 allowed between a struct and one of its base structs or a base struct
37 and one of its children. Otherwise an error is raised.
38 
39 Neither <data> nor <template> will be changed in this process - the
40 result is a new struct value. The actual value of <template> does not
41 matter, only its type.
42 
43 Since the returned struct can't be known at compile time, the
44 efun is declared to return 'mixed'.
45 
46 Introduced in LDMud 3.3.250 .
47 LDMud 3.3.344 added the template argument.
48 LDMud 3.3.433 added the conversion from mappings.
49 LDMud 3.3.720 added the conversion of structs into another struct.
