2  /**
3  * The given array, mapping or struct \p data is returned as a struct. If a \p template struct is given, the returned struct is of the same type. Without a template, an anonymous struct is returned in case of arrays and mappings and in case of structs \p data is returned unchanged.
4  *
5  * If \p data is an array, its elements are assigned in order to the resulting struct. For an anonymous struct, all elements of \p data are assigned, for a templated struct only as many as fit into the struct.
6  *
7  * If \p data is a mapping and no template is given, the resulting anonymous struct contains all elements from \p data with a string key; the key name is used as struct member name.
8  *
9  * If \p data is a mapping and a template is given, the struct member names are used as keys for lookups in \p data; the found data is assigned to the struct members.
10  *
11  * If \p data is a struct and a template is given, a struct of the type of template is created and all members from \p data are copied to the new struct, which exist in both structs. This conversion is only allowed between a struct and one of its base structs or a base struct and one of its children. Otherwise an error is raised.
12  *
13  * Neither \p data nor \p template will be changed in this process - the result is a new struct value. The actual value of \p template does not matter, only its type.
14  *
15  * Since the returned struct can't be known at compile time, the efun is declared to return 'mixed'.
16  *
17  * @synopsis{
18  * mixed to_struct(mixed *|mapping data)
19  * mixed to_struct(mixed *|mapping data, struct template)
20  * mixed to_struct(struct data)
21  * mixed to_struct(struct data, struct template)
22  * }
23  *
24  * @history{
25  * introduced (3.3.250)
26  * changed (3.3.344) -- added the template argument.
27  * changed (3.3.433) -- added the conversion from mappings.
28  * changed (3.3.720) -- added the conversion of structs into another struct.
29  * }
30  *
31  * @see efun::to_array(), efun::to_string(), efun::mkmapping(), @ref driver_syntax_structs "struct"
32  **/
33  mixed to_struct(struct data, struct template);
34 }
