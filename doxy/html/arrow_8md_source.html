1 arrow {#driver_syntax_arrow}
2 ============================
3 The @ref driver_syntax_arrow "arrow" operator (@ref driver_syntax_arrow "->") is equivalent to efun::call_other(). This syntax document discusses how the arrow operator relates to the efun version and gives basic examples; you should see efun::call_other() for thorough documentation.
4 
5 Arrow operator: `->`
6 `ob->fun (mixed arg, ...)`
7 `ob->"fun" (mixed arg, ...)`
8 `ob->(fun) (mixed arg, ...)`
9 
10 `ob->fun(args)` and `ob->"fun"(args)` are equivalent to `call_other(ob, "fun", args)`. `ob->(fun)(args)` is equivalent to `call_other(ob, fun, args)` where fun is a runtime expression returning the function name.
11 
12 @usage{
13 This section explores many ways to call `QueryProp()` in a player object with a single argument, `P_SHORT`. First let's do some setup:
14 
15 ~~~{.c}
16 string str, fun;
17 fun = "QueryProp";
18 
19 ~~~
20 Now we'll use these variables with the basic efun::call_other():
21 
22 ~~~{.c}
23 str = (string)call_other(this_player(), "QueryProp", P_SHORT);
24 str = (string)call_other(this_player(), fun, P_SHORT);
25 
26 ~~~
27 And here are three equivalent calls using the arrow syntax form (with the variables from above):
28 
29 ~~~{.c}
30 str = (string)this_player()->QueryProp(P_SHORT);
31 str = (string)this_player()->"QueryProp"(P_SHORT);
32 
33 str = (string)this_player()->(fun)(P_SHORT);
34 
35 ~~~
36 You can also use the arrow form to call a function on each object in an array, so the following statement calls QueryProp in all interactive users and stores the result in an array of strings:
37 
38 ~~~{.c}
39 string *s;
40 s = (string *)users()->QueryProp(P_SHORT);
41 
42 ~~~
