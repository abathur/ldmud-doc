LPC/pragma.md
1 pragma {#driver_LPC_pragma}
2 ===========================
3 The preprocessor directive #pragma can be used to select
4 several compilation options. Multiple options can be selected
5 in one #pragma directive by separating them with commas.
6 
7 no_clone
8 The blueprint object can't be cloned.
9 
10 no_inherit
11 The program can't be inherited.
12 
13 no_shadow
14 The program can't be shadowed (similar to declaring
15 all functions as 'nomask').
16 
17 init_variables
18 Clone variables are initialized by __INIT().
19 
20 share_variables
21 Clone variables are initialized from the
22 blueprint.
23 
24 weak_types
25 no type checking (this is the default).
26 
27 strict_types
28 all functions must be declared with argument
29 prototypes, and the return values of call_other() must
30 be casted.
31 
32 strong_types
33 all functions must be declared with complete
34 types of return value and parameters.
35 
36 save_types
37 the declaration data is kept after compilation and
38 checked at runtime. This is important for type-safe
39 inheritance.
40 
41 rtt_checks
42 runtime checks during execution of this program will be
43 enabled. The interpreter will check for correct datatypes of
44 arguments on function calls. (Later it will include checks
45 upon assignments.)
46 Don't confuse this with strong/strict_types, they only
47 check at compile time.
48 strong_types/strict_types is seriously recommended.
49 This pragma implicitly enables save_types as well.
50 
51 no_rtt_checks
52 disable runtime type checks for this program (default).
53 
54 pedantic
55 Certain warnings are treated as errors:
56 - failure to pass enough arguments to simul efuns
57 
58 sloppy
59 Turns off pedantic (the default).
60 
61 range_check
62 Use of questionable ranges (ranges of negative sizes,
63 or with bounds beyond the array's size) cause a runtime
64 warning.
65 
66 no_range_check
67 Turns off range_check (the default).
68 
69 warn_deprecated
70 Use of deprecated efuns or indexing operations
71 causes the compiler to issue a warning (the default).
72 
73 no_warn_deprecated
74 Turns off warn_deprecated.
75 
76 warn_empty_casts
77 A cast of a value to its own type generates
78 a warning (the default). Exception are casts to type
79 'mixed'.
80 
81 no_warn_empty_casts
82 Turns off warn_empty_casts.
83 
84 warn_missing_return
85 Warn if a value-returning function is missing
86 a return statement (the default). If possible, the driver
87 will try to detect this at compile time; otherwise a runtime
88 warning will be generated when the function is executed.
89 The check applies only to functions with a declared return
90 type other than 'void'.
91 
92 no_warn_missing_return
93 Turn off warn_missing_return.
94 
95 warn_function_inconsistent
96 If an inherited function is
97 overloaded with inconsistent return types or arguments,
98 a warning is generated; or if pragma_pedantic is in effect,
99 an error. By default this is active.
100 
101 no_warn_function_inconsistent
102 An inherited function can
103 be overloaded with inconsistent return types or arguments,
104 as long as pragma_pedantic is not in effect.
105 
106 This pragma is meant to easen the adaption of legacy
107 mudlib code - in general one should fix the warnings,
108 not turn them off.
109 
110 When an object is compiled with type testing (#pragma
111 strict_types), all types are saved of the arguments for that
112 function during compilation. If the #pragma save_types is
113 specified, then the types are saved even after compilation, to
114 be used when the object is inherited.
115 
116 The following two pragmas are available if the driver was
117 compiled with DEBUG and TRACE_CODE options:
118 
119 set_code_window
120 Sets an offset to the current program writing
121 position. Use this BEFORE a piece of code where you
122 want to check to what bytecodes it is compiled.
123 
124 show_code_window
125 shows some bytes starting at or near the
126 last point set_code_window was called.
