20 Call the function <ob>-><func>() resp. the closure <cl> for every
21 element of the string, array, struct or mapping <arg>, and return a
22 result made up from the returned values.
23 
24 If <ob> is omitted, or neither a string nor an object, it
25 defaults to this_object().
26 
27 If <arg> is an string, array or struct, the function will be called
28 with each of the array/struct elements as first parameter, followed
29 by the <extra> arguments. The result of the efun is an array/struct
30 with all the results returned from the function calls. Thus the
31 operation could be described as:
32 
33  foreach(index) result[index] = ob->func(arg[index], extra...)
34 
35 If <arg> is an string/array/struct, and a mapping is specified
36 instead of a function, the result is an array/struct with the
37 values found in the mapping for the original values, resp. with the
38 original values for which no mapping entry exists. If a column index
39 <idx> is given, that column of the mapping is used. In other words:
40 
41  foreach(index)
42  if (arg[index] exists as key in map)
43  result[index] = map[arg[index]] or map[arg[index], idx]
44 
45  else
46  result[index] = arg[index]
47 
48 If <arg> is a string, the only allowed replacement values are
49 numbers, of which only the lower 8 bit will be considered.
50 
51 If <arg> is a mapping, the function will be called with
52 each of the mapping keys as first, and (if existing) the
53 associated values as second parameter, followed by the <extra>
54 arguments (these must not be protected references like &(i[0]). The
55 result of the efun is a mapping of all results of the function calls,
56 stored under their corresponding key.
57 
58 Depending on the width of the mapping <arg>, the operation can
59 be described as:
60 
61  foreach (key in arg)
62  switch (widthof(arg))
63  case 0:
64 
65  result[key] = ob->func(key, 0, extra...)
66 
67  case 1:
68  result[key] = ob->func(key, arg[key], extra...)
69 
70  else :
71  result[key] = ob->func( key
72  , ({ arg[key,0] ...arg[key,width-1] })
73  , extra...)
74 
75 The advantage of this approach is that the two types of
76 multi-dimensional mappings (mappings with multiple values
77 per key, and mappings of arrays) can be treated in the same way.
78 
79 Note however that the resulting mapping always has one value
80 per key.
81 
82 Historical Note: map() used with arrays behaves like map_array(),
83 but used with mappings generalises map_indices()!
84 
85 EXAMPLES
86 arr = ({ 1, 2, 3, 4 });

88 
89 map(arr, #'%, 2) --> returns ({ 1, 0, 1, 0 })
