12 # Prerequisites #
13 
14  - a POSIX.1-2001 conformant platform
15  - a C99 compatible build environment
16  - bison (yacc won't work! bison 2.7.x is known to work)
17  - supported architectures: i386, x86_64 (work in progress, see section about
18  64 bit systems). Other platforms are not actively supported and tested
19  by us, but many will most likely work as well. Please try and tell us
20  about your experiences.
21 
22  - autoconf and automake for compiling sources directly from our git
23  repository.
24 
25 
26  Optional libraries:
27  - libpcre for the support of PCRE compatible regular expressions
28  - mysql for the mysql package
29  - postgresql for the postgresql package
30  - libsqlite3 for the sqlite package
31  - libgcrypt for various optional algorithms for hashing and crypting
32  - OpenSSL or GnuTLS for TLS support
33  - libxml2 or libiksemel for XML support
34  - json-c (libjson0) for JSON support

36 # Unix or Unix-like system #
37 
38  The driver uses a standard autoconfiguration system which on most
39  systems does all the work for you (for exceptions see below).
40  To prepare the compilation, execute the 'configure' script from
41  within the src/ directory.
42  [Note: If you got LDMud from our git repository please execute the
43  autogen.sh script in src/ to generate the configure script and other
44  auto-generated files.]
45 
46  If you have git or compile for other people (e.g. to distribute a binary),
47  it is seriously recommended to compile from a cloned git repository.
48  Otherwise the version information of the binary will not be precise.
49 
50  configure checks for a number of site specific settings and uses this
51  information to create the files machine.h (from machine.h.in), Makefile
52  (from Makefile.in) and config.h (from config.h.in). We'll come back to
53  config.h below.
54 
55  configure takes a lot of arguments (--help will tell you everything),
56  but the most important are these:
57 
58  --prefix=PREFIX: the base directory for the mud installation,
59  defaults to /usr/local/mud .
60 
61  --bindir=DIR: the directory to install the executables in,
62  defaults to ${PREFIX}/bin .
63 
64  --libdir=DIR: the directory where the mudlib is found,
65  defaults to ${PREFIX}/lib .
66 
67  --includedir=DIR: the directory where driver's LPC include files
68  are supposed to live.
69  defaults to ${PREFIX}/include (which is usually wrong).
70 
71  --libexecdir=DIR: the directory where the programs for the ERQ are found,
72  defaults to ${PREFIX}/libexec .
73 
74  These settings are written into the Makefile and compiled into the driver,
75  just the mudlib directory setting can be changed with a commandline
76  argument.
77 
78  A lot of the drivers parameters can be tweaked for better performance; these
79  parameters are defined in config.h . This file too is created by configure,
80  which provides sensible defaults for all parameters for which no explicite
81  setting is provided. To tweak a setting yourself, pass the argument
82  '--enable-<option>=yes|no' resp. '--with-<option>=<value>' to configure on
83  the commandline.
84 
85  Alternatively, the indivial specifications can be collected in a settings
86  file, which is stored in the directory src/settings/. To use the
87  setting file <osb>, give '--with-<osb>' as argument to configure. The
88  file src/settings/default documents the available settings. The setting
89  files are self-executing: './settings/<foo> [<extra-configure-args>]' will
90  start configure with the proper commandline arguments.
91 
92  The following environment variables can be used to tweak the behaviour
93  of the configure script:
94 
95  CC: the name of the C compiler
96  CFLAGS: compiler flags to be used during the configure script
97  EXTRA_CFLAGS: compiler flags to be used when compiling the game driver
98  LDFLAGS: linker flags to be used by configure and for linking the
99 
100  game driver.
101 
102  After configuration is finished, you may want to modify the Makefile
103  to fine tune those parameters which are not covered by the configuration.
104 
105  The compilation is done using make. Following targets are implemented:
106 
107  <none>: compile the driver, named 'ldmud'.
108  install: compile the driver and install it in ${bindir}
109  utils: compile the utilities, especially the ERQ demon
110  install-utils: compile and install the utilities in ${bindir}
111  install-headers: install the driver header files in ${includedir}.
112  install-all: compile and install everything.
113 
114  To actually run a mud, you need a mudlib. The driver comes with the
115  source of the old 2.4.5 mudlib in mud/lp-245/, support for other mudlibs
116  is also found in mud/. Copy all mudlib files into your 'lib' directory
117  and make sure that the files from mudlib/sys/ all go into your lib's
118  include directory (usually 'sys/').
119 
120  After compilation, you may run our testsuite (work in progress!) in the
121  test subdirectory (launch test/run.sh).
122 
123  To test the driver, start it with 'ldmud 4242'. If you see the message
124  'LDMud ready for users', the driver is up and accepting connections. Test
125  it with 'telnet localhost 4242'.

127 # 64 bit systems (using the LP64 model) #
128 
129 The driver can be compiled on systems which use the LP64 data model (longs
130 and pointers are 64 bit wide). This is the case on many unixoids (but not
131 Windows).
132 There are some muds, which use the driver on LP64 architectures.
133 However, it still has some issues on these architectures. For details
134 please have a look at our bug tracker (@subpage driver_http:__mantis.ldmud.eu_mantis_ "http://mantis.ldmud.eu/mantis/")
135 in the category 'portability' (and there probably are some unknown ones
136 as well).
137 Full support for LP64 machines is planned in the current development
138 series 3.5.x. If you are willing to test and report issues, please do
139 so. ;-)
140 
141 WARNING: LPC savefiles from 64 bit systems with integers exceeding the
142  (signed) 32 bit integers can't be restored on 32 bit systems!
143 
144 If you want to compile the driver for a 64 bit platforms and your compiler
145 does not produce LP64 code by default, we suggest to set CFLAGS, LDFLAGS
146 and EXTRA_CFLAGS (and maybe CC) prior to configure to suitable values to
147 cause your compiler to compile for the desired platform, e.g.
148 # export CFLAGS="-m64"
149 # export LDFLAGS="-m64"
150 # export EXTRA_CFLAGS=$CFLAGS
151 # ./configure
152 
153 Setting EXTRA_FLAGS is important, otherwise the driver will be compiled using
154 different compiler options that the ones used during the configure run.
155 It is crucial that you DON'T CHANGE the compiler options controlling the
156 platform in the generated Makefile, unless you know very well what you
157 do. (If the machine.h generated by configure is for a different platform, the
158 driver may assume wrong sizes of data types, resulting in undefined behaviour
159 and crashes.)
160

192 # IPv6 #
193 
194  If your machine supports IPv6, the driver can be configured to use it: give
195  '--enable-use-ipv6=yes' as argument to the configure script.
196 
197 # mySQL #
198 
199  If your machine has mySQL installed, the driver can be configured to use
200  it: give '--enable-use-mysql=yes' as argument to the configure script.
201 
202  Alternatively, if your mySQL uses an unusual include/library path,
203  the option can be given as '--enable-use-mysql=/unusual/path', which
204  will use the given path as search path for both include and library
205  files in addition to the normal system search paths. The include files will
206  be searched in <path>/include and <path>, the library files will be search
207  in <path>/lib/mysql, <path>/lib, and <path>, in this order.
208 
209  The username and password for the mySQL database are specified by
210  the mudlib as arguments to the efun db_connect().
211 
212  Use mysqladmin to create any databases you want to provide - the
213  names are later used in the efun db_connect() to connect to
214  the databases.

216 # Windows 95/98/NT/XP/7 #
217 
218  To compile the gamedriver for Windows, you need the 'Cygwin' package,
219  which is a port of gcc, bash, and other GNU/Unix programs. Once it
220  is installed and running, the procedure is the same as under Unix.
221 
222  CygWin is available from @subpage driver_http:__cygwin.com_ "http://cygwin.com/".
223  (When installing make sure that your installation includes gcc, bash, make,
224 
225  sed, awk, and bison and stuff listed above...)
226 
227  There are some pre-compiled binaries for Windows linked from our homepage.
228  Please have a look at:
229  @subpage driver_http:__www.ldmud.eu_resources.html "http://www.ldmud.eu/resources.html"
230 
231  However, they are compiled for specific mudlibs, so your mileage may vary.
