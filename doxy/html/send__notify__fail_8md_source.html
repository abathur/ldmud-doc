11 
12 Optional hook to send the notify fail message, regardless of how it was determined, to the player. If the hook is not set, the message is delivered using efun::tell_object() internally. Hook setting can be a string or a closure.
13 
14 If the hook is a string, it is the name of a (possibly static) function to call in the current command giver. If the hook is a closure, it is the function to be called. Lambda closures are bound to the current command giver first.
15 
16 The arguments to the call are:
17 - <msg> is the notify fail message to be delivered.
18 - <msgobj> is the object which set the message. It is 0 for the default message.
19 - <orig_cmd_giver> is the object for which the original command was first received. It is usually identical with the current command giver `this_player()`.
