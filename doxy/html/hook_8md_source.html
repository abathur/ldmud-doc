1 hook {#driver_hook_hook}
2 ========================
3 To allow a greater flexibility of the muds, the gamedrivers since 3.2.1 moved several once hardcoded 'underground' activities from the driver into the mudlib. This includes for example the differences between compat and native mode.
4 
5 The hooks are set with the privileged efun set_driver_hook(). Some of the hooks are mandatory, some not. Most hooks accept unbound lambda closures as values, some also lfun closures or even strings.
6 
7 The hooks are identified by an ordinal number, for which symbolic names are defined in `/sys/driverhooks.h`.
8

10 @ref driver_hook_move_object "H_MOVE_OBJECT1"
11 Mandatory hooks to implement the efun void move_object().

15 Mandatory hooks to determine the uid and euid of loaded or cloned objects.

22 @ref driver_hook_create_super "H_CREATE_SUPER" is called for blueprints implicitely loaded by inheritance, @ref driver_hook_create_ob "H_CREATE_OB" for explicitely loaded blueprints/objects, and @ref driver_hook_create_clone "H_CREATE_CLONE" for cloned objects.

33 @ref driver_hook_default_prompt "H_DEFAULT_PROMPT"
34 Optional hook for the command prompt. If this hook is not used, the driver will use "> " as the command prompt.
35 
36 @ref driver_hook_print_prompt "H_PRINT_PROMPT"
37 Optional hook to print the current command prompt. If this hook is not set, the driver will just print the prompt to the user.
38 
39 @ref driver_hook_command "H_COMMAND"
40 Optional hook to parse and execute commands. If this hook is used, it bypasses the normal command parsing done by the driver (including the MODIFY_COMMAND and NOTIFY_FAIL hooks).
41 
42 @ref driver_hook_modify_command "H_MODIFY_COMMAND"
43 Optional hook to modify commands (both entered or given by a call to command()) before the parser sees them (this includes special commands like 'status').
44 
45 @ref driver_hook_modify_command_fname "H_MODIFY_COMMAND_FNAME"
46 Mandatory hook specifying the name of the 'modify_command' lfun to call for newly entered commands as result of a set_modify_command().
47 
48 @ref driver_hook_notify_fail "H_NOTIFY_FAIL"
49 Mandatory hook to issue the default message if an entered command couldn't be parsed and no notify_fail() command is in effect.
50 
51 @ref driver_hook_send_notify_fail "H_SEND_NOTIFY_FAIL"
52 Optional hook to send the notify fail message, regardless of how it was determined, to the player. If the hook is not set, the message is delivered using tell_object() internally.
53 
54 @ref driver_hook_no_ipc_slot "H_NO_IPC_SLOT"
55 Optional hook specifying the message given to logins rejected due to space limitations (MAX_PLAYER).
56 
57 @ref driver_hook_include_dirs "H_INCLUDE_DIRS"
58 Semi-mandatory hook specifying the directories where <>-type include files are searched (this includes ""-includes not found as specified).
59 
60 @ref driver_hook_auto_include "H_AUTO_INCLUDE"
61 Optional hook specifying a string to be included before the source of every compiled LPC object.
62 
63 @ref driver_hook_telnet_neg "H_TELNET_NEG"
64 Optional hook to specifiy how to perform a single telnet negotiation. If not set, most telnet options are rejected (read: only a very minimal negotiation takes place).
65 
66 @ref driver_hook_no_echo "H_NOECHO"
67 Optional hook to specifiy how to perform the telnet actions to switch the echo mode (used for e.g. password input_to()s). If not set, a default handling is performed.
68 
69 IMPORTANT: If this hook is used, the control of all telnet negotiation is transferred to the mudlib (you must combine it with @ref driver_hook_telnet_neg "H_TELNET_NEG" to conform to the telnet protocol).
70 
71 @ref driver_hook_erq_stop "H_ERQ_STOP"
72 Optional hook to notify the mudlib about the termination of the erq demon.
73 
74 @ref driver_hook_msg_discarded "H_MSG_DISCARDED"
75 Optional hook to specify a message or take other measures when a message had to be discarded.
