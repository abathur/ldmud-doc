SYNOPSIS¶

¶
void efun308( object item, object env)

DESCRIPTION¶

The item is moved into its new environment env, which may be 0. This efun is to be used in the move_object() hook, as it does nothing else than moving the item - no calls to init() or such.

Don’t use it in your own objects!

HISTORY¶

  • introduced (3.2.1@1)
  • changed (3.2.6) – renamed to set_environment()(E)
  • changed (3.2.9) – not available if driver is compiled without USE_DEPRECATED.
  • removed (3.3)
